## 1. Вступ

### Що таке SushiChain?

SushiChain — це повноцінна платформа для прийому та обробки замовлень їжі. Вона дозволяє вам:

- **Керувати меню** — додавати, редагувати, ховати позиції, налаштовувати опції (розміри, топінги).
- **Приймати замовлення** — відстежувати статуси, призначати кур'єрів, експортувати дані.
- **Налаштовувати доставки** — зони, тарифи, мінімальні суми замовлення.
- **Створювати акції** — знижки, промокоди, безкоштовна доставка, бонусні бали.
- **Аналізувати продажі** — дашборд з графіками, топ-продукти, сегменти клієнтів.
- **Спілкуватися з клієнтами** — модерація відгуків, обробка скарг, email/Telegram кампанії.
- **Керувати кількома філіями** з єдиної адмін-панелі.

### Як отримати доступ до адмін-панелі

1. Відкрийте браузер (Chrome, Safari, Edge, Firefox).
2. Перейдіть за адресою вашого сайту, додавши `/admin` наприкінці. Наприклад: `https://vash-restoran.com/admin`.
3. Увійдіть, використовуючи ваші облікові дані адміністратора (email та пароль), які вам надали під час реєстрації.
4. Після входу ви побачите головну сторінку аналітики з навігаційним меню зліва.

> **На скріншоті видно:** Ліва бічна панель із логотипом бренду «Суші Мастер» та навігаційними пунктами: Аналітика, Бренди, Філії, Зони доставки, Меню, Замовлення, Акції, Відгуки, Фідбек, Кампанії, Розширена аналітика, Кур'єри, Онбординг, Налаштування бренду, Юридичні документи.

### Огляд навігації

| Розділ | Що робить |
|---|---|
| **Аналітика** | Головний дашборд: доходи, кількість замовлень, середній чек, графіки |
| **Бренди** | Керування брендом (назва, кольори, лого, слоган) |
| **Філії** | Додавання та налаштування ресторанів/відділень |

| **Акції** | Промокоди та автоматичні знижки |
| **Відгуки** | Модерація та відповіді на відгуки клієнтів |

| **Кампанії** | Email/Telegram розсилки |

## 2. Перше налаштування

### Запуск майстра онбордингу

Після першого входу в адмін-панель перейдіть до розділу **Онбординг** у лівому меню (іконка ракети 🚀). Майстер онбордингу проведе вас через кілька кроків:

1. **Основна інформація про бренд** — назва, логотип, кольори, слоган.
2. **Створення першої філії** — адреса, телефон, графік роботи.
3. **Налаштування зон доставки** — визначення зон та тарифів.
4. **Додавання категорій меню** — базові категорії.
5. **Додавання перших продуктів** — назви, ціни, описи.

> **На скріншоті видно:** Крок онбордингу з формою для введення назви бренду, вибору основного кольору через колірний пікер, поля для слогану та посилання на логотип.

### Поради щодо заповнення інформації про бренд

| Поле | Порада |
|---|---|
| **Назва** | Використовуйте реальну назву вашого ресторану. Вона відображатиметься у заголовку сайту та в додатку. |
| **Slug** | Автоматично генерується з назви (транслітерація з української). Це частина URL-адреси. Наприклад, для «Суші Мастер» буде `sushi-master`. Можете змінити вручну. |
| **Основний колір** | Виберіть колір, який асоціюється з вашим брендом. Використовується для кнопок, акцентів та заголовків на сайті. |
| **Додатковий колір** | Другорядний колір для фонів та декоративних елементів. Підбирайте так, щоб добре поєднувався з основним. |
| **Акцентний колір** | Колір для особливих елементів: бейджі, індикатори, спеціальні пропозиції. |
| **Слоган** | Коротка фраза, яка відображається на головній сторінці. Наприклад: «Найкращі роли у місті». |
| **Логотип** | Вкажіть URL-адресу зображення вашого логотипу. Рекомендований розмір — не менше 200×200 пікселів, формат PNG або SVG з прозорим фоном. |
| **Валюта** | За замовчуванням — UAH (гривня). Якщо ви працюєте в іншій валюті, змініть код та символ. |
| **Опис** | Детальний опис вашого ресторану для сторінки «Про нас». |

### Після онбордингу

Після завершення всіх кроків майстра:

1. Перевірте, як виглядає ваш сайт, відкривши головну сторінку.
2. Переконайтеся, що кольори та логотип відображаються коректно.
3. Додайте більше продуктів у меню (див. [розділ 3](#3-управління-меню)).
4. Налаштуйте графік роботи філій (див. [розділ 4](#4-філії-та-доставка)).

## 3. Управління меню

### 3.1. Категорії меню

Категорії допомагають організувати ваші продукти. Наприклад: «Ролі», «Сети», «Wok», «Напої», «Десерти».

#### Як створити категорію

1. У лівому меню розгорніть **Меню** → виберіть **Категорії**.
2. Натисніть кнопку **«Створити категорію»** у правому верхньому куті.
3. Заповніть форму:
   - **Назва** — обов'язкове поле. Наприклад: «Роли».
   - **Slug** — генерується автоматично, але можна змінити вручну.
   - **Опис** — коротке пояснення категорії (необов'язково).
   - **Зображення** — URL-адреса картинки для категорії (необов'язково).
   - **Порядок сортування** — число, що визначає позицію категорії в меню. Менші значення — вище в списку.
   - **Активна** — увімкніть, щоб категорія відображалася на сайті.
4. Натисніть **«Зберегти»**.

> **На скріншоті видно:** Таблиця зі списком категорій: «Роли», «Сеты», «Wok», «Напитки» тощо. Кожна категорія має назву, slug, кількість продуктів, статус та кнопки редагування/видалення.

#### Як змінити порядок категорій

Змініть поле **«Порядок сортування»** (sort order) у формі редагування категорії. Категорії з меншим значенням відображаються першими. Рекомендуємо використовувати кроки по 10 (10, 20, 30), щоб було зручно вставляти нові категорії між існуючими.

#### Як приховати категорію

Увімкніть або вимкніть перемикач **«Активна»**. Прихована категорія не відображається на сайті клієнтів, але її продукти залишаються в базі даних.

### 3.2. Продукти

#### Як додати продукт

1. Перейдіть до **Меню** → **Продукти**.
2. Натисніть **«Додати продукт»**.
3. Заповніть форму:

| Поле | Опис |
|---|---|
| **Категорія** | Виберіть категорію зі списку (обов'язково). |
| **Філія** | Залиште порожнім, якщо продукт доступний у всіх філіях. Або виберіть конкретну філію. |
| **Назва** | Назва продукту, наприклад «Філадельфія класика» (обов'язково). |
| **Slug** | Генерується автоматично. |
| **Опис** | Детальний опис: склад, смакові якості. |
| **Зображення** | URL-адреса фотографії продукту. Рекомендований розмір — 400×400 пікселів. |
| **Ціна** | Базова ціна у гривнях (обов'язково). |
| **Вага** | Наприклад: «350г». |
| **Калорії** | Калорійність в кілокалоріях. |
| **Вегетаріанське** | Позначте, якщо страва підходить вегетаріанцям. |
| **Доступний** | Перемикач наявності. Якщо вимкнено — продукт не відображається на сайті (немає в наявності). |
| **Теги** | Ключові слова для пошуку через кому: `роли,сети,вегетарианское`. |
| **Алергени** | Перелік алергенів через кому: `глютен,молоко,орехи`. |
| **Порядок сортування** | Число для визначення позиції в межах категорії. |

4. Натисніть **«Зберегти»**.

> **На скріншоті видно:** Форму створення продукту з усіма полями: випадаючий список категорій, поле назви, поле ціни з символом ₴, перемикач «Доступний», текстові поля для опису та тегів.

#### Опції продукту (розміри, топінги, dodatki)

Опції дозволяють клієнтам налаштовувати продукт під себе. Наприклад: розмір порції, рівень гостроти, додаткові інгредієнти.

**Як додати групу опцій:**

1. У формі редагування продукту знайдіть розділ **«Групи опцій»**.
2. Натисніть **«Додати групу»**.
3. Заповніть:
   - **Назва групи** — наприклад «Розмір» або «Додаткові топінги».
   - **Обов'язкова** — якщо увімкнено, клієнт зобов'язаний обрати хоча б одну опцію.
   - **Макс. вибір** — скільки опцій може обрати клієнт (1 = лише одна, 0 = необмежено).
   - **Порядок сортування** — позиція групи в списку.
4. Додайте опції всередині групи:
   - **Назва** — наприклад «Велика +80₴».
   - **Доплата** — додаткова сума до базової ціни.
   - **Порядок сортування** — позиція опції в групі.

**Приклад налаштування розмірів піци:**

| Назва | Доплата | Порядок |
|---|---|---|
| Маленька (25 см) | 0 ₴ | 1 |
| Середня (35 см) | +60 ₴ | 2 |
| Велика (45 см) | +120 ₴ | 3 |

#### Масові дії з продуктами

На сторінці продуктів ви можете:

- **Виділити кілька продуктів** — поставте прапорці біля потрібних продуктів. Зверху з'явиться панель масових дій.
- **Приховати виділені** — натисніть іконку ❌ (PackageX), щоб швидко позначити обрані продукти як недоступні.
- **Показати виділені** — натисніть іконку ✅ (PackageCheck), щоб повернути продукти в меню.
- **Видалити виділені** — натисніть іконку кошика для видалення (з підтвердженням).

#### Фільтрація та пошук продуктів

- **Пошук** — введіть назву продукту в поле пошуку.
- **Фільтр за категорією** — виберіть категорію зі списку.
- **Фільтр наявності** — «Усі», «В наявності» або «Немає в наявності».

## 4. Філії та доставка

### 4.1. Додавання нової філії

1. Перейдіть до **Філії** у лівому меню.
2. Натисніть **«Створити філію»**.
3. Заповніть форму:

| Поле | Опис |
|---|---|
| **Назва** | Наприклад: «Суші Мастер — Хрещатик» (обов'язково). |
| **Slug** | Генерується автоматично з назви. |
| **Адреса** | Повна адреса: «вул. Хрещатик, 22, Київ» (обов'язково). |
| **Телефон** | Контактний номер філії. |
| **Email** | Email філії (необов'язково). |
| **Широта / Довгота** | Координати для відображення на карті. |
| **Опис** | Додаткова інформація про філію. |
| **Зображення** | URL-адреса фотографії філії. |

4. Натисніть **«Зберегти»**.

> **На скріншоті видно:** Таблиця філій із колонками: Назва, Місто (визначається з адреси), Адреса, Телефон, Автопідтвердження, Прийом замовлень, Статус, Дії. Кожна філія має кнопки редагування, активації/деактивації та видалення.

### 4.2. Графік роботи

Графік роботи вказується у форматі JSON у полі **«Робочий графік»**. Формат:

```json
{
  "mon": "09:00-22:00",
  "tue": "09:00-22:00",
  "wed": "09:00-22:00",
  "thu": "09:00-22:00",
  "fri": "09:00-23:00",
  "sat": "10:00-23:00",
  "sun": "10:00-21:00"
}
```

- Дні тижня: `mon`, `tue`, `wed`, `thu`, `fri`, `sat`, `sun`.
- Час у форматі `ГГ:ХХ-ГГ:ХХ` (24-годинний формат).
- Якщо філія не працює в певний день, вкажіть порожнє значення: `"mon": ""`.

> **Порада:** Рекомендуємо вказувати реальний час роботи, щоб клієнти бачили коректну інформацію під час оформлення замовлення.

### 4.3. Зони доставки

Зони доставки визначають, куди ви доставляєте та за яку ціну.

1. Перейдіть до **Зони доставки** у лівому меню.
2. Натисніть **«Створити зону»**.
3. Заповніть:

| Поле | Опис |
|---|---|
| **Назва** | Назва зони: «Центр», «Лівий берег», «Ближнє передмістя». |
| **Філія** | Виберіть філію, до якої належить зона. |
| **Опис** | Додатковий опис зони (необов'язково). |
| **Мін. сума замовлення** | Мінімальна сума для доставки в цю зону. |
| **Вартість доставки** | Фіксована вартість доставки. |
| **Час доставки (хв)** | Очікуваний час доставки в хвилинах. |
| **Активна** | Увімкніть, щоб зона була доступна для клієнтів. |

4. Натисніть **«Зберегти»**.

> **На скріншоті видно:** Таблиця зон доставки з колонками: Назва, Філія, Мін. сума, Вартість, Час, Статус. Рядки кольорово кодовані: активні зони — зеленим, неактивні — сірим.

**Приклад конфігурації зон:**

| Зона | Філія | Мін. сума | Доставка | Час |
|---|---|---|---|---|
| Центр (3 км) | Хрещатик | 200 ₴ | 0 ₴ | 30 хв |
| Ближнє передмістя (5 км) | Хрещатик | 300 ₴ | 50 ₴ | 45 хв |
| Далеке передмістя (10 км) | Хрещатик | 500 ₴ | 100 ₴ | 60 хв |

### 4.4. Прийом замовлень та автопідтвердження

У формі редагування філії є два важливі перемикачі:

- **Прийом замовлень** (acceptingOrders) — швидко вимкніть/увімкніть прийом замовлень для філії. Коли вимкнено, клієнти не зможуть створити замовлення з цієї філії.
- **Автопідтвердження** (autoConfirm) — коли увімкнено, нові замовлення автоматично переходять у статус «Підтверджено» без ручного підтвердження. Коли вимкнено — кожне замовлення потребує ручного підтвердження оператором.

> **Порада:** Для нових філій рекомендуємо спочатку вимкнути автопідтвердження, щоб контролювати кожне замовлення. Після налагодження процесів можна увімкнути.

### 4.5. Мінімальна сума та час підготовки

- **Мінімальна сума замовлення** (minOrderAmount) — встановлюється на рівні філії. Замовлення з меншою сумою не будуть прийняті.
- **Час підготовки** (prepTimeMinutes) — за замовчуванням 30 хвилин. Це орієнтовний час, який показується клієнту при оформленні замовлення.

## 5. Акції та промокоди

### 5.1. Типи акцій

SushiChain підтримує чотири типи акцій:

| Тип | Опис | Приклад |

| **Відсоток** (percentage) | Знижка у відсотках від суми замовлення | 20% знижки |

| **Значення** | Величина знижки: для відсотка — `20`, для фіксованої суми — `50`, для бонусу — `100`. |

Також відображаються: кількість використань, дати дії, кнопки редагування та видалення.

## 6. Замовлення

### 6.1. Перегляд списку замовлень

1. Перейдіть до **Замовлення** у лівому меню.
2. На сторінці відображається таблиця всіх замовлень із фільтрами.

> **На скріншоті видно:** Таблиця замовлень із колонками: № замовлення, Клієнт, Філія, Тип (доставка/самовивіз), Статус (кольоровий бейдж), Сума, Дата. Зверху — фільтри за статусом, філією, датою та поле пошуку.

**Доступні фільтри:**

- **За статусом** — Усі, Нові, Підтверджені, Готуються, Готові, Доставляються, Виконані, Скасовані.
- **За філією** — виберіть конкретну філію.
- **За датою** — вкажіть період «від» та «до».
- **Пошук** — пошук за номером замовлення або ім'ям клієнта.

### 6.2. Статуси замовлень та їхні переходи

Кожне замовлення проходить через визначені статуси:

```
Нове → Підтверджене → Готується → Готове → Доставляється → Виконане
  ↓        ↓           ↓          ↓           ↓
Скасовано  Скасовано   Скасовано  Скасовано  Скасовано
```

| Статус | Опис | Що означає для клієнта |
|---|---|---|
| **Нове** (new) | Замовлення щойно створено | «Замовлення отримано» |
| **Підтверджене** (confirmed) | Ресторан прийняв замовлення | «Замовлення підтверджено, починаємо готувати» |
| **Готується** (cooking) | Замовлення в процесі приготування | «Ваше замовлення готується» |
| **Готове** (ready) | Замовлення готове до видачі/відправки | «Замовлення готове» |
| **Доставляється** (delivering) | Кур'єр забрав замовлення | «Кур'єр в дорозі» |
| **Виконане** (completed) | Замовлення доставлено/видано | «Дякуємо за замовлення!» |
| **Скасоване** (cancelled) | Замовлення скасовано | «Замовлення скасовано» |

### 6.3. Зміна статусу замовлення

1. Натисніть на іконку **👁 (перегляд)** біля замовлення.
2. У вікні деталей замовлення виберіть новий статус із випадаючого списку.
3. Натисніть **«Оновити статус»**.

> **Важливо:** Перехід між статусами відбувається строго послідовно. Ви не можете перевести замовлення з «Нового» одразу в «Готується» — спочатку потрібно підтвердити.

**Як скасувати замовлення:**

Будь-яке замовлення (крім «Виконаного» та «Скасованого») можна скасувати. Натисніть кнопку скасування у вікні деталей. Система запитає підтвердження.

### 6.4. Призначення кур'єрів

Щоб призначити кур'єра на доставку:

1. Перейдіть до розділу **Кур'єри**.
2. Знайдіть кур'єра та натисніть іконку 📦 (Package) біля його імені.
3. У вікні виберіть замовлення зі статусом «Готове» або «Доставляється».
4. Натисніть **«Призначити»**.

> **На скріншоті видно:** Список кур'єрів із колонками: Ім'я, Телефон, Активних замовлень, Всього доставок, Статус. Поруч із кожним кур'єром — кнопки редагування, призначення доставки та перегляду історії доставок.

### 6.5. Деталі замовлення

У вікні перегляду замовлення ви побачите:

- **Номер замовлення** (наприклад: #1042).
- **Дані клієнта** — ім'я, телефон, адреса доставки.
- **Список позицій** — продукти, кількість, обрані опції, ціни.
- **Підсумки** — підсумок, знижка, вартість доставки, бонуси, загальна сума.
- **Спосіб оплати** — картка, готівка, бонуси (кольоровий бейдж).
- **Статус оплати** — очікує, обробляється, успішно, невдало, повернуто.
- **Часова шкала** — коли створено, підтверджено, почато приготування тощо.
- **Коментар клієнта** — примітки до замовлення (наприклад: «без кунжуту»).

### 6.6. Експорт замовлень у CSV

1. На сторінці замовлень натисніть кнопку **«Завантажити CSV»** (іконка ⬇ Download).
2. Файл CSV буде завантажений automatically.
3. Файл містить усі замовлення з урахуванням встановлених фільтрів.

> **Порада:** Використовуйте експорт CSV для аналізу в Excel або Google Таблицях. Встановіть потрібний період у фільтрах перед експортом, щоб отримати лише необхідні дані.

## 7. Відгуки та фідбек

### 7.1. Модерація відгуків

Клієнти можуть залишати відгуки після отримання замовлення. Всі відгуки спочатку мають статус **«На модерації»** (не затверджені) і не відображаються на сайті, поки ви їх не затвердите.

1. Перейдіть до **Відгуки** у лівому меню.
2. У списку ви побачите червоний лічильник із кількістю відгуків, що очікують модерації (також відображається в бічному меню).
3. Для кожного відгуку доступні дії:
   - ✅ **Затвердити** — відгук з'явиться на сайті.
   - ❌ **Видалити** — відгук буде остаточно видалено.

> **На скріншоті видно:** Таблиця відгуків із колонками: Дата, Клієнт, Продукт, Замовлення, Оцінка (зірки), Коментар, Статус модерації, Відповідь адміна, Дії. Невідмодеровані відгуки виділені жовтим фоном.

### 7.2. Відповідь на відгук

Ви можете відповісти на відгук клієнта:

1. Натисніть іконку **💬 (повідомлення)** біля відгуку.
2. У вікні введіть вашу відповідь.
3. Натисніть **«Надіслати відповідь»**.

Ваша відповідь відобразиться під відгуком на сайті. Це показує клієнтам, що ви дбаєте про зворотний зв'язок.

> **Порада:** Завжди відповідайте на негативні відгуки. Вибачтеся за незручності та запропонуйте рішення. Це показує іншим клієнтам, що ви відповідальний бізнес.

### 7.3. Обробка фідбеку (скарг та пропозицій)

Розділ **Фідбек** містить повідомлення клієнтів різних типів:

| Тип | Опис | Колір бейджу |
|---|---|---|
| **Проблема з замовленням** (order_issue) | Скарга щодо конкретного замовлення | Червоний |
| **Загальне питання** (general) | Загальне питання або коментар | Сірий |
| **Пропозиція** (suggestion) | Ідея для покращення | Синій |
| **Скарга** (complaint) | Офіційна скарга | Оранжевий |

**Статуси обробки фідбеку:**

| Статус | Опис |
|---|---|
| **Новий** (new) | Повідомлення щойно надійшло |
| **В обробці** (in_progress) | Ви працюєте над вирішенням |
| **Вирішено** (resolved) | Проблему вирішено |
| **Закрито** (closed) | Обробку завершено |

**Як відповісти на фідбек:**

1. Натисніть на повідомлення у списку.
2. У вікні перегляду прочитайте повідомлення клієнта, його контактні дані та прив'язане замовлення (якщо є).
3. Введіть вашу відповідь у поле **«Відповідь адміністратора»**.
4. Змініть статус на **«В обробці»**, **«Вирішено»** або **«Закрито»**.
5. Натисніть **«Зберегти»**.

> **На скріншоті видно:** Детальне вікно фідбеку з інформацією про клієнта, типом проблеми (червоний бейдж «Проблема з замовленням»), текстом повідомлення, полем для відповіді та кнопками зміни статусу.

### 7.4. Поради щодо роботи зі скаргами

1. **Реагуйте швидко** — намагайтеся відповідати на скарги протягом 1-2 годин.
2. **Будьте ввічливими** — навіть якщо клієнт незадоволений, відповідайте професійно.
3. **Пропонуйте рішення** — компенсація, безкоштовна доставка наступного замовлення, знижка.
4. **Відстежуйте тенденції** — якщо однакові скарги повторюються, це сигнал про системну проблему.
5. **Змінюйте статуси** — це допомагає не загубити жодне повідомлення.

## 8. Аналітика

### 8.1. Головний дашборд

Головна сторінка адмін-панелі («Аналітика») містить основні показники вашого бізнесу.

> **На скріншоті видно:** Чотири картки метрик у верхній частині: «Замовлення» (значення + тренд), «Дохід» (значення у ₴ + тренд), «Середній чек» (значення + тренд), «Активні акції» (кількість + тренд). Нижче — два графіки: лінійний графік замовлень по днях та стовпчиковий графік доходу за категоріями. В самому низу — таблиця останніх замовлень.

#### Метрики

| Метрика | Опис |
|---|---|
| **Замовлення** | Загальна кількість замовлень за обраний період. |
| **Дохід** | Загальна сума всіх виконаних замовлень за період. |
| **Середній чек** | Середня сума одного замовлення (дохід ÷ кількість замовлень). |
| **Активні акції** | Кількість активних промоакцій у поточний момент. |

Біля кожної метрики відображається **тренд** — відсоток зміни порівняно з попереднім періодом. Зелена стрілка вгору — зростання, червона стрілка вниз — падіння.

#### Періоди

Виберіть період для аналізу, натиснувши відповідну кнопку:

- **Сьогодні**
- **Останні 7 днів** (за замовчуванням)
- **Останні 30 днів**
- **Цей місяць**
- **Довільний період** — вкажіть дати «від» та «до».

### 8.2. Графіки

**Замовлення по днях (лінійний графік):**

Показує кількість замовлень за кожен день обраного періоду. Допомагає побачити:
- Дні найбільшого та найменшого попиту.
- Тенденції зростання або падіння.
- Вплив акцій та свят на кількість замовлень.

**Дохід за категоріями (стовпчиковий графік):**

Показує, які категорії меню приносять найбільше доходу. Кожна категорія має свій колір. Допомагає:
- Визначити найприбутковіші категорії.
- Зрозуміти, які категорії потребують просування.

### 8.3. Таблиця останніх замовлень

У нижній частині дашборду — таблиця з 5 останніми замовленнями:

| Колонка | Опис |

### 8.4. Розширена аналітика

Перейдіть до **Розширена аналітика** у лівому меню для детальнішої інформації.

#### Сегменти клієнтів

| Сегмент | Опис |
|---|---|
| **Нові клієнти** | Клієнти з 1 замовленням |
| **Повертаються (2-5)** | Клієнти з 2-5 замовленнями |
| **Лояльні (6+)** | Клієнти з 6 та більше замовленнями |
| **VIP (висока ціна)** | Клієнти з високим середнім чеком |

> **На скріншоті видно:** Таблиця сегментів клієнтів із колонками: Сегмент (з кольоровою крапкою), Кількість клієнтів, Загальна сума покупок, Середній чек, Відсоток від загальної кількості. Кожен сегмент має індивідуальний колір фону.

#### Замовлення за годинами

Горизонтальний стовпчиковий графік, що показує розподіл замовлень по годинах доби (00:00 – 23:00). Допомагає:
- Визначити **пік-години** — час найбільшого навантаження.
- Оптимізувати графік роботи персоналу.
- Правильно розподілити зміни кухарів та кур'єрів.

#### Замовлення за днями тижня

Графік, що показує, які дні тижня найпопулярніші для замовлень. Допомагає планувати запаси та персонал.

#### Воронка конверсії

| Етап | Опис |
|---|---|
| **Кошики** | Кількість створених кошиків |
| **Оформлення** | Кількість переходів до оплати |
| **Виконані** | Кількість успішно виконаних замовлень |

Конверсія розраховується як відсоток переходу між етапами. Якщо конверсія з кошиків в оформлення низька — можливо, проблема у процесі оплати або вартості доставки.

#### Показник повторних замовлень (Repeat Rate)

Відсоток клієнтів, які зробили більше ніж одне замовлення. Високий показник свідчить про лояльність клієнтів.

## 9. Кампанії

### 9.1. Що таке кампанії?

Кампанії — це цілеспрямовані розсилки повідомлень вашим клієнтам через **Telegram** або **Email**. Ви можете створювати:

- **Win-back** — повернення неактивних клієнтів.
- **Промо** — інформування про акції та спеціальні пропозиції.
- **Сповіщення** — важливі повідомлення (зміна графіка, нові продукти).

### 9.2. Створення кампанії

1. Перейдіть до **Кампанії** у лівому меню.
2. Натисніть **«Створити кампанію»**.
3. Заповніть форму:

| Поле | Опис |
|---|---|
| **Назва** | Внутрішня назва кампанії. |
| **Тип** | `win_back`, `promo` або `notification`. |
| **Канал** | `telegram` або `email`. |
| **Цільовий сегмент** | Виберіть аудиторію (див. нижче). |
| **Тема** | Тема повідомлення (для email) або заголовок. |
| **Тіло повідомлення** | Текст повідомлення. Можна використовувати змінні: `{name}` — ім'я клієнта, `{bonusBalance}` — баланс бонусів. |
| **Статус** | `draft` (чернетка), `active` (активна), `paused` (призупинена), `completed` (завершена). |

4. Натисніть **«Зберегти»**.

> **На скріншоті видно:** Таблиця кампаній із колонками: Назва, Тип (кольоровий бейдж), Канал, Сегмент, Статус, Надіслано/Відкрито/Клікнуто, Дії. Для кожної кампанії доступні кнопки: редагування, відправка (іконка ✉ Send), видалення.

### 9.3. Цільові сегменти

| Сегмент | Опис |
|---|---|
| **all** | Усі клієнти бренду |
| **new** | Нові клієнти (1 замовлення) |
| **inactive_7d** | Неактивні більше 7 днів |
| **inactive_14d** | Неактивні більше 14 днів |
| **inactive_30d** | Неактивні більше 30 днів |
| **high_value** | Клієнти з високим середнім чеком (VIP) |

### 9.4. Відправка кампанії

1. Створіть кампанію та збережіть її (статус — «Чернетка»).
2. Перевірте текст та налаштування.
3. Натисніть іконку **✉ (Send)** біля кампанії.
4. Підтвердіть відправку.

Після відправки статус зміниться на **«Активна»**, а потім на **«Завершена»**.

### 9.5. Перегляд результатів кампанії

Для кожної кампанії відображаються метрики:

| Метрика | Опис |
|---|---|
| **Надіслано** (sentCount) | Кількість відправлених повідомлень |
| **Відкрито** (openedCount) | Кількість відкритих повідомлень |
| **Клікнуто** (clickedCount) | Кількість переходів за посиланнями |

> **Порада:** Відстежуйте **відсоток відкриття** (opened ÷ sent × 100) та **відсоток кліків** (clicked ÷ opened × 100), щоб оцінювати ефективність кампаній.

## 10. Налаштування бренду

### 10.1. Що таке фічі-прапорці (feature flags)?

Фічі-прапорці — це перемикачі, які дозволяють вмикати або вимикати окремі функції вашого ресторану на сайті. Ви можете гнучко налаштувати, які функції доступні клієнтам.

Перейдіть до **Налаштування бренду** (іконка ⚙ Settings) у лівому меню.

> **На скріншоті видно:** Сторінку налаштувань із перемикачами, згрупованими за категоріями: «Замовлення», «Клієнтські функції», «Оплата», «Лояльність». Кожен перемикач має назву функції та короткий опис.

### 10.2. Опис усіх функцій

| Перемикач | За замовчуванням | Опис |
|---|---|---|
| **Замовлення без реєстрації** (enableGuestOrders) | ✅ Увімкнено | Дозволяє клієнтам робити замовлення без створення акаунта |
| **Заплановані замовлення** (enableScheduledOrders) | ✅ Увімкнено | Дозволяє клієнтам обирати час доставки/самовивізу |
| **Програма лояльності** (enableLoyalty) | ✅ Увімкнено | Включає систему бонусних балів |
| **Відгуки** (enableReviews) | ✅ Увімкнено | Дозволяє клієнтам залишати відгуки |
| **Доставка** (enableDelivery) | ✅ Увімкнено | Увімкнення опції доставки |
| **Самовивіз** (enablePickup) | ✅ Увімкнено | Увімкнення опції самовивозу |
| **Акції** (enablePromotions) | ✅ Увімкнено | Дозволяє використовувати промокоди |
| **Рекомендації** (enableRecommendations) | ✅ Увімкнено | Показує рекомендації «Вам може сподобатися» |
| **Пошук** (enableSearch) | ✅ Увімкнено | Показує рядок пошуку в меню |
| **Обране** (enableFavorites) | ✅ Увімкнено | Дозволяє додавати продукти в улюблені |
| **Вимагати телефон** (requirePhone) | ✅ Увімкнено | Телефон обов'язковий при оформленні |
| **Вимагати адресу** (requireAddress) | ✅ Увімкнено | Адреса обов'язкова при доставці |
| **Чайові** (showTips) | ❌ Вимкнено | Показує можливість залишити чайові кур'єру |
| **Apple Pay** (enableApplePay) | ✅ Увімкнено | Дозволяє оплату через Apple Pay |
| **Функції доставки** (enableDeliveryFeatures) | ✅ Увімкнено | Розширені налаштування доставки |
| **Кампанії** (enableCampaigns) | ✅ Увімкнено | Дозволяє створювати розсилки |
| **Переклади меню** (enableTranslations) | ❌ Вимкнено | Увімкнення багатомовного меню (укр/ру/ен) |

### 10.3. Налаштування програми лояльності

У розділі налаштувань бренду також знаходяться параметри лояльності:

| Параметр | За замовчуванням | Опис |
|---|---|---|
| **Коєфіцієнт лояльності** (loyaltyRate) | 5 | Кількість балів за кожні 100 ₴ витрат. Наприклад, при `5` за замовлення на 500 ₴ клієнт отримає 25 балів. |
| **Мін. сума для нарахування** (loyaltyMinSpend) | 0 ₴ | Мінімальна сума замовлення, за яку нараховуються бали. |
| **Макс. оплата бонусами** (loyaltyMaxBonusPayment) | 50 ₴ | Максимальна сума, яку можна оплатити бонусами за одне замовлення. |

### 10.4. Кастомізація оформлення замовлення

За допомогою фічі-прапорців ви можете налаштувати процес оформлення замовлення:

- **Вимагати телефон** — якщо вимкнено, клієнти можуть замовити без вказання телефону (для замовлень без реєстрації).
- **Вимагати адресу** — якщо вимкнено, адреса необов'язкова (корисно для самовивозу).
- **Чайові** — увімкніть, якщо хочете дати клієнтам можливість залишити чайові кур'єру.
- **Заплановані замовлення** — дозволяє клієнтам обрати зручний час доставки.

> **Порада:** Якщо ви працюєте лише з самовивозом, вимкніть «Доставка» та «Вимагати адресу». Це спростить процес оформлення для клієнтів.

**Що має містити політика конфіденційності:**

## 12. Часті питання

### Як змінити кольори бренду?

1. Перейдіть до **Бренди** у лівому меню.
2. Натисніть іконку **✏ (редагувати)** біля вашого бренду.
3. У формі редагування знайдіть поля кольорів:
   - **Основний колір** — кольоровий пікер та текстове поле з HEX-кодом.
   - **Додатковий колір** — другорядний колір.
   - **Акцентний колір** — колір для акцентів.
4. Виберіть новий колір за допомогою пікера або введіть HEX-код вручну (наприклад: `#e11d48`).
5. Натисніть **«Зберегти»**.

Зміни кольорів будуть миттєво відображені на сайті клієнтів.

> **Порада:** Використовуйте інструменти на кшталт [coolors.co](https://coolors.co) для підбору гармонійної кольорової палітри. Основний колір має забезпечувати достатній контраст з білим текстом на кнопках.

---

### Як скинути або оновити меню?

**Щоб видалити всі продукти та почати заново:**

1. Перейдіть до **Меню** → **Продукти**.
2. Виберіть усі продукти, поставивши прапорець у заголовку таблиці.
3. На панелі масових дій натисніть кнопку видалення (🗑).
4. Підтвердіть видалення.

> **Увага:** Видалення продуктів є незворотнім. Рекомендуємо експортувати дані перед такими змінами. Продукти, що входять до існуючих замовлень, залишаються в історії замовлень.

**Щоб оновити ціни масово:**

Наразі ціни оновлюються індивідуально для кожного продукта через форму редагування. Для масового оновлення рекомендуємо:
1. Відфільтрувати продукти за категорією.
2. Відкривати кожен продукт та оновлювати ціну.
3. Використовувати поле «Порядок сортування» для організації.

---

### Як обробити невдалий платіж?

Якщо клієнт скаржиться на невдалу оплату:

1. Відкрийте **Замовлення** та знайдіть замовлення клієнта.
2. Перевірте **статус оплати** в деталях замовлення:
   - **Очікує** (pending) — платіж ще не оброблено. Попросіть клієнта почекати або спробувати ще раз.
   - **Обробляється** (processing) — платіж в процесі. Зазвичай завершується протягом кількох хвилин.
   - **Невдало** (failed) — платіж не пройшов. Клієнт має спробувати інший спосіб оплати.
   - **Повернуто** (refunded) — кошти повернуто клієнту.
3. Якщо платіж у статусі «Невдало», клієнт може повторити замовлення.
4. Якщо проблема системна (багато невдалих платежів), зверніться до технічної підтримки.

> **На скріншоті видно:** Деталі замовлення з виділеним блоком оплати: спосіб оплати (бейдж «Картка»), статус оплати (червоний бейдж «Невдало»), сума оплати, ID транзакції платіжної системи.

### Як зв'язатися з технічною підтримкою?

Якщо у вас виникли технічні проблеми або питання щодо роботи платформи:

1. **Email** — надішліть лист на адресу підтримки, вказану в вашому договорі.
2. **Telegram** — зверніться до вашого персонального менеджера в Telegram.
3. **Через розділ «Фідбек»** — створіть повідомлення з типом «Загальне питання».

**Що вказати в запиті:**

- Опис проблеми (що саме не працює).
- Кроки для відтворення проблеми (що ви робили перед тим, як проблема виникла).
- Скріншоти з помилками (якщо є).
- Номер браузера та операційної системи.
- Час виникнення проблеми.

> **Порада:** Перед зверненням до підтримки спробуйте оновити сторінку (Ctrl+F5) або очистити кеш браузера. Багато проблем вирішуються таким чином.
